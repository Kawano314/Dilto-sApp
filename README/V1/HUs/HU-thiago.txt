História 1: "Como cliente, quero reservar uma mesa de sinuca pelo app, para garantir disponibilidade ao chegar."

Critérios de aceitação:
1. O sistema deve mostrar os horários livres para reserva.
2. Deve ser possível selecionar um horário e confirmar a reserva.

---

História 2: "Como dono da conveniência, quero limitar o número de reservas, para evitar conflitos de disponibilidade."

Critérios de aceitação:
1. O sistema deve bloquear automaticamente reservas acima da quantidade de mesas.
2. O dono deve conseguir liberar uma reserva manualmente, caso necessário.

---

História 3: "Como cliente, quero ser avisado se minha mesa está disponível ou ocupada, para não perder tempo ao chegar."

Critérios de aceitação:
1. O sistema deve exibir o status de cada mesa em tempo real.
2. Se a reserva for confirmada, deve aparecer como “Reservada”.
3. O status deve ser atualizado automaticamente quando a reserva expira ou é finalizada.
